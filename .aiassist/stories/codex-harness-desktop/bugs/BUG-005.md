# BUG-005: TopBar 主题/语言切换存在竞态，偶发切换失败

> story: `codex-harness-desktop`  
> status: `fixed`  
> classification: `code-defect`  
> created_at: 2026-07-10  
> reporter: agent（全量回归发现）

---

## 症状

全量 E2E 回归时，`topbar.test.cjs` 中 `theme toggle in topbar switches data-theme` 偶发失败：点击 TopBar 主题按钮后 `data-theme` 未发生变化。

## 复现步骤

1. 运行 `npm run test:e2e`。
2. 当 `topbar.test.cjs` 与 `onboarding.test.cjs` 等测试并发或按不同顺序执行时，TopBar 自身的 `useSettings` 可能尚未返回 settings。
3. 此时 `theme` 取默认值 "dark"，若实际 `document.documentElement.dataset.theme` 已被 App 加载为 "light"，点击按钮会尝试将主题设为 "light"，看起来没有变化。

## 根因

`TopBar.jsx` 的 `toggleTheme` / `toggleLanguage` 依赖组件内部的 `settings` state 来判断当前值。`TopBar` 与 `App` 各自调用 `useSettings`，存在加载时间差；在 settings 返回前，TopBar 使用默认值（theme="dark"），可能与 DOM 上已应用的实际值不一致。

## 修复

将 `toggleTheme` / `toggleLanguage` 改为优先读取 `document.documentElement` 上已应用的 `data-theme` / `lang` 属性作为当前值，再计算下一个值。这样即使 `settings` state 尚未加载，切换逻辑也能与当前实际状态保持一致。

## 关联需求

- **REQ-I18N-001**：主题切换。
- **REQ-I18N-002**：语言切换。
