# BUG-005: TopBar 主题/语言切换存在竞态，偶发切换失败

> story: `codex-harness-desktop`  
> status: `fixed`  
> classification: `code-defect`  
> created_at: 2026-07-10  
> reporter: agent（全量回归发现）

---

## 症状

全量 E2E 回归时，`topbar.test.cjs` 中 `theme toggle in topbar switches data-theme` 偶发失败：点击 TopBar 主题按钮后 `data-theme` 未发生变化或断言因初始值为 `null` 崩溃。

## 复现步骤

1. 运行 `npm run test:e2e`。
2. 当 `topbar.test.cjs` 在页面尚未完成 settings 加载就开始断言时，`html[data-theme]` 可能尚未设置。
3. `TopBar` 与 `App` 各自调用独立的 `useSettings()` hook，存在状态不同步；TopBar 的 toggle 可能基于过时的默认值。

## 根因

- `TopBar.jsx` 的 `toggleTheme` / `toggleLanguage` 依赖组件内部的 `settings` state 来判断当前值。
- `TopBar` 与 `App` 各自调用 `useSettings()`，拥有独立的 state 和更新闭包；在 settings 返回前，TopBar 使用默认值，可能与 DOM 上已应用的实际值不一致。
- `useSettings` 的 `applyToDocument` 在 settings 字段缺失时不设置默认值，导致首次启动时 `html[data-theme]` 可能为 `null`。

## 修复

1. **共享 settings 状态**：将 `useSettings` 重构为 React Context（`SettingsProvider`），`App` 与 `TopBar` 共享同一份 settings 状态和更新函数，消除多 hook 实例之间的竞争。
2. **基于共享 state 切换**：`TopBar.jsx` 的 `toggleTheme` / `toggleLanguage` 直接从共享的 `settings` 读取当前值，计算并调用 `updateSettings({ theme: next })` / `updateSettings({ language: next })`。
3. **保证 DOM 默认值**：`useSettings` 的 `applyToDocument` 对 `theme`、`density`、`language` 均设置回退默认值，确保 `html[data-theme]` 等属性始终存在。
4. **稳定测试断言**：`topbar.test.cjs` 在读取 `data-theme` 前先 `await expect(...topbar).toBeVisible()`，确保 settings 已加载；并对 `getAttribute` 结果提供 `"dark"` 回退，避免 `null` 导致 `toHaveAttribute` 内部错误。
