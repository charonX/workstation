# BUG-003: Project 创建表单中 "Local Path" 标签未国际化

> story: `codex-harness-desktop`  
> status: `fixed`  
> classification: `code-defect`  
> created_at: 2026-07-10  
> reporter: user (feel-signoff)

---

## 症状

在 `ProjectFormModal.jsx` 中，本地项目路径输入框的标签硬编码为英文 `"Local Path"`，未通过 `t(...)` 走 i18n，语言切换到 `zh-CN` 时该标签仍显示英文。

## 复现步骤

1. 启动应用并进入 Workspace 页面。
2. 点击 "Add Project" 打开项目创建弹层。
3. 切换语言到中文。
4. 观察 Local Path 输入框标签。

## 期望行为

"Local Path" 标签应根据当前语言显示为 "Local Path"（en-US）或 "本地路径"（zh-CN），与 `projectDetail.localPath` 的翻译一致。

## 实际行为

标签始终显示英文 "Local Path"。

## 环境

- branch: `story/skill-modal`
- 路径：`src/renderer/components/project/ProjectFormModal.jsx:89`
- 关联 i18n key：`projectDetail.localPath` 已存在翻译

## 关联需求

- **REQ-I18N-002**：界面支持 `zh-CN` / `en-US` 切换，所有用户可见文案通过 i18n key 管理。

## 修复记录

- 在 `src/renderer/i18n/en-US.json` 和 `zh-CN.json` 中新增 `projectForm.*` 翻译键。
- 将 `src/renderer/components/project/ProjectFormModal.jsx` 中所有硬编码英文（标题、来源、项目名称、Local Path、仓库地址、分支、按钮）替换为 `t(...)`。
- Onboarding E2E 测试通过，未破坏现有流程。
