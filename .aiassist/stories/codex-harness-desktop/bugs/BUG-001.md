# BUG-001: 顶部导航栏内容与 UX HTML 不一致

> story: `codex-harness-desktop`  
> status: `fixed`  
> classification: `code-defect` (with test-gap)  
> created_at: 2026-07-10  
> fixed_at: 2026-07-10  
> reporter: user (feel-signoff)

---

## 症状

应用顶部导航栏（TopBar）在实现中仅渲染了一个空壳，缺少 UX HTML 中定义的品牌 Logo、全局搜索框以及右侧功能图标，导致 feel-signoff 时与参照原型明显不一致。

## 复现步骤

1. 启动 Electron 应用（`npm run dev` 或 E2E 启动）。
2. 进入任意页面（Dashboard / Workspace / Flows / Tasks / Skills / Settings）。
3. 观察页面顶部通栏。

## 期望行为（基于 UX HTML）

参照 `.aiassist/stories/codex-harness-desktop/ux/dashboard-overview.html` 与 `workspace-home.html` 中的 `.topbar` 结构：

- 左侧：品牌文字 **"OPC Workstation"** + 全局搜索框（placeholder: "Search projects, flows, tasks..."）。
- 右侧：四个图标按钮，依次为 **语言切换**、**主题切换**、**通知**、**设置**。
- 整体样式：高度、`--ch-topbar-height`、背景 `--ch-surface`、底部边框 `--ch-border`、flex 左右分布。

用户提供的截图（UX HTML）明确显示了上述全部元素。

## 实际行为

- `src/renderer/components/layout/TopBar.jsx` 仅返回空的 `topbar-left` 与 `topbar-right` div，没有任何内容。
- 运行中的产品顶部通栏为空白，仅有背景色和边框。

## 关联需求

- **PRD §6.4 关键页面结构**：所有主要页面均基于当前交互原型，TopBar 作为全局布局元素出现在每个 UX HTML 原型中。
- **REQ-DASH-001**：UX 参照 `dashboard-overview.html`，该文件顶部包含上述 TopBar 结构。
- **REQ-WORKSPACE-005**：项目搜索/过滤功能在 PRD 中定义；TopBar 的全局搜索框是该能力的全局入口（与 Workspace 页面过滤形成互补）。
- **REQ-I18N-001 / REQ-I18N-002**：语言切换与主题切换入口当前位于 Settings 页面侧边栏导航中，而 UX 原型要求其在 TopBar 右上角提供快捷入口。

## 根因（初步）

实现阶段 `TopBar.jsx` 被创建为占位组件，未按 UX HTML 填充内容。该组件被 `PageLayout.jsx` 全局引用，因此影响所有页面。

## 分类与路由

| 维度 | 结论 | 依据 |

| code-defect | ✅ | 实现偏离 UX HTML 这一 feel-signoff 参照标准；`TopBar.jsx` 空实现导致页面结构缺失。 |
| test-gap | ✅ | 现有 E2E 测试未覆盖 TopBar 元素存在性（locators.cjs 中无 TopBar 相关定位器），回归测试需补充。 |
