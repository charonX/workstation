# BUG-004: local path 输入框应使用目录选择器而非纯文本输入

> story: `codex-harness-desktop`  
> status: `triaged`  
> classification: `req-gap`（功能增强）  
> created_at: 2026-07-10  > reporter: user (feel-signoff)

---

## 症状

项目中所有需要选择本地目录的输入框（项目本地路径、Workspace 根目录、Skill 仓库路径）目前都是纯文本输入框，用户必须手动键入完整路径，易出错且体验差。

## 复现步骤

1. 打开 Settings 页面：Workspace Root Directory、Skill Repository Path 为文本输入。
2. 打开 Workspace → Add Project：选择本地项目时 Local Path 为文本输入。

## 期望行为

这些字段旁提供目录选择按钮，点击后弹出系统目录选择器（`electron.dialog.showOpenDialog`），用户选择目录后自动回填路径到输入框。

## 实际行为

只能手动输入路径。

## 环境

- branch: `story/skill-modal`
- 涉及路径：
  - `src/renderer/pages/Settings.jsx`（workspaceRoot、skillRepoPath）
  - `src/renderer/components/project/ProjectFormModal.jsx`（localPath）
- 需要改动：
  - `src/preload/preload.js` 暴露目录选择 API
  - `src/main/main.js` 注册 IPC handler

## 关联需求

- **REQ-WORKSPACE-001**：Workspace 根目录配置。
- **REQ-WORKSPACE-002**：Skill 仓库位置配置。
- **REQ-WORKSPACE-003**：添加本地项目。
- **PRD §4.12 项目导入 UI**：支持本地目录导入。

当前 REQ 只要求"可配置/可输入"，未明确输入方式。目录选择器是更合理的 UX 实现，属于需求细化。
