# BUG-004: local path 输入框应使用目录选择器而非纯文本输入

> story: `codex-harness-desktop`  
> status: `fixed`  
> classification: `acceptable-deviation`（可接受偏差）  
> created_at: 2026-07-10  
> reporter: user (feel-signoff)

---

## 症状

项目中所有需要选择本地目录的输入框（项目本地路径、Workspace 根目录、Skill 仓库路径）此前都是纯文本输入框，用户必须手动键入完整路径，易出错且体验差。

## 复现步骤

1. 打开 Settings 页面：Workspace Root Directory、Skill Repository Path 为文本输入。
2. 打开 Workspace → Add Project：选择本地项目时 Local Path 为文本输入。

## 期望行为

这些字段旁提供目录选择按钮，点击后弹出系统目录选择器（`electron.dialog.showOpenDialog`），用户选择目录后自动回填路径到输入框。

## 实际行为（修复后）

- Settings 页面的 Workspace Root Directory 与 Skill Repository Path 已替换为 `DirectoryInput`，点击 Browse 打开系统目录选择器。
- ProjectFormModal 的 Local Path 已替换为 `DirectoryInput`，点击 Browse 打开系统目录选择器。
- 选择目录后路径自动回填到对应输入框。

## 实现范围

- `src/preload/preload.js`
  - 暴露 `window.opc.selectDirectory(title, defaultPath)`。
  - 新增测试钩子 `window.opc.__setSelectDirectoryImpl`，便于 E2E mock 原生对话框。
- `src/main/main.js`
  - 注册 `opc-select-directory` IPC handler，调用 `dialog.showOpenDialog`。
- `src/renderer/components/shared/DirectoryInput.jsx`（新增）
  - 通用组件：文本输入框 + Browse 按钮。
- `src/renderer/pages/Settings.jsx`
  - Workspace Root、Skill Repo Path 使用 `DirectoryInput`。
- `src/renderer/components/project/ProjectFormModal.jsx`
  - Local Path 使用 `DirectoryInput`。
- `src/renderer/i18n/en-US.json` / `zh-CN.json`
  - 新增 `common.browse` 翻译键。
- `src/renderer/index.css`
  - 新增 `.directory-input` 样式。
