# BUG-002: 应用整体布局结构错误 — TopBar 未跨顶且存在两个 Logo

> story: `codex-harness-desktop`  
> status: `fixed`  
> classification: `code-defect`

> fixed_at: 2026-07-10  
> reporter: user (feel-signoff)

## 症状

BUG-001 修复后，页面布局仍与 UX HTML 不一致：

1. 当前实现是**左右结构**（Sidebar 占左侧全高，右侧区域再分上下放 TopBar + 内容）。
2. UX HTML 是**上下结构**（TopBar 横跨整个顶部，下方再分左右：Sidebar + Main）。
3. 当前 Sidebar 中存在一个 `sidebar-logo`，TopBar 中也有一个 `topbar-logo`，导致页面上出现两个 "OPC Workstation" 品牌标识。

## 复现步骤

1. 启动 Electron 应用。
2. 进入任意页面。
3. 观察左侧边栏顶部和顶部通栏：各有一个 "OPC Workstation" 品牌。
4. 对比 `.aiassist/stories/codex-harness-desktop/ux/dashboard-overview.html` 与 `workspace-home.html` 的布局结构。

## 期望行为（基于 UX HTML）

参照 UX HTML：

- `.app-shell`：`grid-template-columns: sidebar-width 1fr; grid-template-rows: topbar-height 1fr;`
- `.topbar`：`grid-column: 1 / -1`（跨顶部全宽）。
- `.sidebar`：位于 TopBar 下方左侧，**没有自己的品牌 Logo**。
- `.main` / `.app-content`：位于 TopBar 下方右侧。

## 实际行为

- `PageLayout.jsx` 使用 `Sidebar` + `app-main`（`TopBar` 在 `app-main` 内部），形成左右结构。
- `index.css` 中 `.sidebar` 占 `grid-row: 1 / -1`，`.topbar` 只占 `grid-row: 1; grid-column: 2`。
- `Sidebar.jsx` 包含 `sidebar-logo`。
- 页面上同时出现 Sidebar Logo 和 TopBar Logo。

## 关联需求

- **PRD §6.4 关键页面结构**：所有主要页面基于同一套交互原型，全局布局为 TopBar 跨顶 + Sidebar + Main。
- **REQ-DASH-001**：UX 参照 `dashboard-overview.html` 要求此布局结构。
- **REQ-I18N-001 / REQ-I18N-002**：主题/语言切换入口位于 TopBar 右上角，不应在 Sidebar 中重复出现品牌。

## 根因

BUG-001 修复时只关注了 TopBar 内容填充，未重构 `PageLayout` 的 CSS Grid 结构，也未移除 Sidebar 中的 Logo，导致整体布局与 UX HTML 不一致。
